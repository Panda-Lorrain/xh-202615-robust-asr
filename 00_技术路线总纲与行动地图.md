## 〇、一句话定位

> 给定**目标说话人的唤醒音频**，在「带噪 + 多人重叠」的识别音频中，**只转写目标、拒识非目标**，且要在 L20 GPU 上跑得快。

这不是普通的语音识别，本质是 **Target-Speaker ASR（TS-ASR）+ 意图拒识** 的复合任务。

---

## 一、题目逐字拆解（任务边界）

| 题目要素 | 技术含义 | 难点 / 考点 |
|---|---|---|
| **唤醒音频（enrollment）** | 目标说话人的参考语音，长度未知（可能极短，<0.5s） | 超短 enrollment → 声纹不稳定；对应美的 #3 US-PVAD 的 0.2s 考点 |
| **识别音频** | 待处理的混合音频 | 含噪声 + 干扰人，目标指令埋在其中 |
| **SNR −5~5 dB** | 极低信噪比，噪声远大于普通场景 | −5dB 是"几乎听不清"级别；对应 #8 VPIDM 扩散增强考点 |
| **多说话人重叠 ≤2 人** | 目标 + 至多 1 个干扰人 | 重叠率 0–100% 连续分布，100% 是完全同说 |
| **重叠率 0–100%** | 从干净到全重叠都要处理 | 不能假设"总能分离"，需要帧级判别 |
| **只转写目标** | TS-ASR，输出仅目标 transcript | 声纹条件化 ASR（DiCoW 范式） |
| **拒识非目标** | 干扰人说的话不转写、且要"拒绝" | 不仅是过滤，是**主动拒识**（40% 独立评分，对应 #1 LLM 拒识） |

> ⚠️ **关键边界澄清**：「拒识」在本题是**独立评分维度（40%）**，不只是"别把干扰人的话转写出来"。美的 #1 论文表明：拒识要判断「这条指令在智能家居场景是否合理/是否针对当前设备」，是**语义层 + 声纹层**的联合判断，不是单纯的声纹阈值过滤。

## 二、评分 → 技术模块 映射（40/40/20 怎么拿）

| 评分维度 | 权重 | 核心技术模块 | 主力论文资产 |
|---|---|---|---|
| **目标 CER** | **40%** | TS-ASR 转写（DiCoW/SE-DiCoW）+ 中文领域微调 + 后处理纠错 | DiCoW 系列、#6 RASTAR |
| **拒识率** | **40%** | LLM 语义拒识 + 声纹置信度 + Personal VAD 检测 | #1 Reject-or-Not、#2/#3 PVAD |
| **推理效率** | **20%** | 流式/轻量架构（TS-RNNT 形态）+ Whisper 蒸馏/量化 | TS-RNNT、Whisper-turbo |

> **战略含义**：CER 和拒识各占 40%，**不能偏科**。一个 CER 极低但拒识差、或拒识好但 CER 崩的方案都不行。效率 20% 是"锦上添花但不可忽视"——同分情况下效率决定排名。

- M2：US-PVAD（DPRNN）/ ASE-PVAD（CAM++）

- M4：DiCoW / SE-DiCoW（github BUTSpeechFIT/TS-ASR-Whisper + HF `BUT-FIT/SE_DiCoW`）
- M5：RASTAR 思路（BERT 改写 + 检索 + CoT）
- M6：Qwen-2.5-3B（开源）+ #1 拒识框架

## 四、通道数分水岭：双预案（⚠️ 待确认，当务之急）

> 测试集是**单通道**还是**多通道**，决定空间/DOA 路线能否启用。**这是唯一卡住全局架构的未知项。**

### 预案 A：单通道（组合主线直接用）
- 空间路线（DSENet/KWS空间/VSAEC）**全部不可用**（它们需要 ≥2~3 麦克风）。
- 组合主线（上图）**完整可用**——它本来就是单通道友好的。
- 前端降噪：改用单通道增强（VPIDM 扩散 / 常规降噪），或纯靠 Whisper + 数据增强的鲁棒性。

### 预案 B：多通道（组合主线 + 空间前端增量）
- 组合主线**仍然完整可用**（多通道只是额外加分）。
- 在 M2 之前**加空间前端**：
  - **DSENet / VSAEC**（DOA 引导、几何无关 beamforming）做目标方向语音提取
  - **KWS 空间先验**（#3）用 DOA 辅助定位目标说话人
- 多通道能显著提升 CER（尤其在重叠 + 远场），是**潜在的冲上限增量**。
- ⚠️ **空间路线可行性诚实评估**（47-agent critic 指出的最大盲区，答辩必问）：DSENet/VSAEC **无预训练权重、未测 −5dB、仅 6 人重叠 3 通道圆阵验证**，需从零复现（高风险）；且 **DOA 在完全重叠区（两人同向）同样有歧义**，空间路线解决不了 100% 重叠。故多通道是"与 PVAD 路线互补的增量"，**不是银弹、不作主力押注**；若确认多通道，DSENet 作可选前端，主力仍是通道无关的组合主线。详见 `03_答辩FAQ与风险预案.md` 第四节。

> **关键判断**：**无论通道数如何，组合主线都是下限保障**。所以不要干等通道数——先跑通组合主线；多通道只是"确认后叠加的前端增强"。等待期的所有 W1-W7 工作在两种预案下都通用。

## 五、差异化策略矩阵（凭什么赢别的队）

> 诚实前提：组合主线用的都是公开 + 开源技术，**强队也能找到 DiCoW**。不能靠"用了 DiCoW"赢。壁垒在下面 5 个维度。

| # | 差异化点 | 拿的分 | 壁垒（别人为何难复制） | 优先级 |
|---|---|---|---|---|
| **D1** | **数据增强做到极致**：仿真贴真实分布 + 覆盖最广（噪声/混响/语速/信道） | CER | 脏活累活，要时间+工程 | 🔴 最高 |
| **D2** | **中文家居领域微调**：指令语料、家居实体词、方言 | CER | 中文领域语料 + 微调 know-how | 🔴 最高 |
| **D3** | **端到端联合训练**（PVAD+ASR+拒识，反 cascaded） | CER+拒识 | 出题方明确偏好；工程难，壁垒高 | 🔴 高 |
| **D4** | **三路融合拒识**（声纹+语义+VAD）+ 自适应 CoT | 拒识 | 拒识策略创新空间大 | 🟠 高 |
| **D5** | **效率优化**（TS-RNNT 形态 / Whisper 蒸馏 / 量化 / 流式） | 效率 | 工程优化深度 | 🟠 高 |

> **两个"上限赌注"**（见 `02_上限候选深读`）：
> - **候选 X：端到端联合训练**（D3）—— 评委认可、壁垒高，主押。
> - **候选 Y：音频大模型一体化**（Qwen2-Audio 直接吃音频做 TS-ASR+拒识）—— 新颖度最高、风险也最高，作探索分支。

**心法**：比赛不是"技术新颖度"竞赛，是"**完成度 + 适配度 + 效率**"竞赛。把开源 baseline 在中文/家居/极端 SNR/超短 enrollment 上调到极致，比换新模型更容易拉开差距。

## 六、行动地图（W1–W7 + 上限候选）

> 全部标注：**目标 / 通道依赖 / 产出 / 工作量 / 依赖**。优先级 P0→P3。

### 🔴 P0（立即并行启动）

#### W1 — 跑通 SE-DiCoW 开源 baseline
- **目标**：`github.com/BUTSpeechFIT/TS-ASR-Whisper` + HF 权重 `BUT-FIT/SE_DiCoW` 本地跑通；测 Whisper-large-v3-turbo 在中文 + −5dB 的真实表现
- **通道依赖**：❌ 无
- **产出**：一个能推理的 TS-ASR pipeline；知道开源 baseline 的真实上下限
- **工作量**：2–3 天（环境 + 权重 + demo）
- **依赖**：无
- **为何最先做**：地基——所有后续决策都基于"开源 baseline 能打到什么程度"

#### W2 — 数据仿真 pipeline
- **目标**：造符合题目分布的中文仿真集（SNR −5~5dB、重叠 0–100%、≤2 人）
- **数据源**：AISHELL-1/2/3、WenetSpeech、MagicData-RAMC（对话）+ MUSAN 噪声 + 房间脉冲响应（RIR）
- **通道依赖**：❌ 无（单通道仿真先做；多通道预案下再加多通道仿真）
- **产出**：训练集 + 验证集，含 enrollment + 混合音频 + 目标文本 + 重叠 mask
- **工作量**：1 周（脚本 + 参数可配置 + 跑通）
- **依赖**：无
- **设计要求**：仿真参数全可配置（SNR/重叠率/噪声类型/混响），真实数据来了能快速校准

### 🟠 P1（W1/W2 有进展后启动）

#### W3 — Personal VAD 前端
- **目标**：复现 US-PVAD（DPRNN，不依赖 SV）+ ASE-PVAD（CAM++）两条路线
- **通道依赖**：❌ 无
- **产出**：帧级 TSS/NTSS → **STNO mask 生成器**（组合主线的胶水，M3）
- **工作量**：2–3 周（复现 + 训练）
- **依赖**：W2（训练数据）
- **为何重要**：出题方核心偏好；单独也贡献拒识/检测分

#### W5 — LLM 拒识系统
- **目标**：按 #1 论文框架用 Qwen-2.5-3B 做"智能家居指令合理性"拒识 + 自适应 CoT
- **通道依赖**：❌ 无
- **产出**：独立的拒识模块（吃 transcript → 输出 accept/reject）
- **工作量**：2–3 周（部署 + prompt + 指令语料构造）
- **依赖**：无（可独立优化，与 CER/效率解耦）

### 🟡 P2

#### W4 — enrollment 声纹处理
- **目标**：CAM++ 提唤醒音频声纹；处理 0.2s 超短参考场景
- **通道依赖**：❌ 无
- **产出**：目标说话人 embedding 接口（M1）
- **工作量**：3–5 天
- **依赖**：无

#### W7 — 把给定 enrollment 接入 SE-DiCoW
- **目标**：把 SE-DiCoW 的 self-enrollment（自动选片段）改成**接受题目给定的唤醒音频**作 enrollment 条件
- **通道依赖**：❌ 无
- **产出**：SE-DiCoW 真正当 baseline（题目输入约定下）
- **工作量**：1–2 周
- **依赖**：W1（跑通 SE-DiCoW）

### ⚪ P3（贯穿全程）

#### W6 — 评测体系 + 效率测量
- **目标**：CER 工具 + 拒识率指标定义 + 各模块 RTF（real-time factor）测量
- **通道依赖**：❌ 无
- **产出**：可复用的评测脚本；效率基线
- **工作量**：3–5 天（持续维护）
- **依赖**：W1（要有模型才能测 RTF）

### ⛔ 必须等通道数（预案 B 才做）
- DOA/空间前端：DSENet / KWS 空间先验 / VSAEC / beamforming
- 多通道仿真数据生成

### 🎯 上限赌注（见 02 文档）
- **候选 X（端到端联合训练）**：在 W3+W4+W5 成熟后启动，工作量 4–8 周
- **候选 Y（音频大模型一体化）**：并行探索分支，风险高，不押全部身家

## 七、阶段里程碑（建议节奏）

> 假设比赛周期充裕。每阶段有明确可验证的交付。

| 阶段 | 时间 | 交付 | 验证标准 |
|---|---|---|---|

| **阶段 3：差异化** | 第 6–10 周 | D1/D2 数据增强 + 中文微调；D4 拒识融合 | CER 显著降；拒识率达标 |

## 九、待确认事项（行动前必须澄清）

1. ⚠️ **测试集单/多通道**（最关键，决定空间路线）—— 向主办方确认或看 A 集数据
2. **enrollment 格式**：整句唤醒词 / 任意片段？长度范围？（影响声纹策略 + SE-DiCoW 接入）
3. **标注粒度**：字级 / 句级？有无重叠区说话人标注？（影响训练监督信号）
4. **远场 / 近场**：识别音频是否含回声、混响？（决定是否需要 AEC，对应 VSAEC #7）
5. **指令集范围**：智能家居指令的领域边界？（影响 LLM 拒识的"合理性"判别标准）

> 前 4 项影响架构，建议**第一时间向主办方确认**或从 A 集数据中分析。第 5 项影响拒识，需从题目/数据中归纳。
