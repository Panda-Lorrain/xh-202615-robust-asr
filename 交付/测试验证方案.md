# 测试验证方案

> 对齐官方评分(目标 CER 40% / 拒识率 40% / 推理效率 20%,L20 GPU)。
> 当前结果为**仿真 450 条 + target 缺席 72 条**,真实 A 集到手后按 §4 流程重测。

---

## 1. 评测指标(对齐评分维度)

| 评分维度 | 权重 | 本项目指标 | 计算方式 |
|---|---|---|---|
| 目标 CER | 40% | 字符级 CER(jiwer) | `eval_metrics.py:cer`;**拒识条计 CER=1.0**(漏 target) |
| 拒识率 | 40% | target-absent 集真实拒识率(RR)+ target-present 集误拒率(FRR) | RR=正确拒/应拒总数;FRR=误拒/应转总数 |
| 推理效率 | 20% | overall_rtf + 峰值显存 | `timing.json:overall_rtf`(L20 上重测) |

**关键约定**:
- CER 计算时,**被系统拒识的 target-present 条**计 CER=1.0(等于完全漏转)→ 拒识太严会同时伤 CER 和 FRR。
- 拒识率口径:**target-absent 集真实拒识率(越高越好)** + **target-present 集误拒率(越低越好)**,两者需同时报。

---

## 2. 评测脚本

| 脚本 | 用途 |
|---|---|
| `code/eval_metrics.py` | 单次评测:CER / RTF / 拒识(precision/recall/f1/reject_rate)/ batch |
| `code/fuse_eval.py` | **多策略融合扫最优工作点**(评测用,非推理):读 enroll_infer JSON + LLM verdicts + manifest,扫 `sim_threshold × strategy` 排序 |
| `code/build_reject_set.py` | 造 target-absent 集(72 条,真实拒识率评测用) |
| `code/eval_se_cer.py` | SE baseline vs denoised CER 对比 |
| `code/eval_enrollment.py` | wespeaker 分档指标(overlap/SNR) |

**融合策略**(`submit_infer.py --strategy` / `fuse_eval.py`):
- `sim_only`:拒 iff `max_sim < sim_thr`
- `llm_only`:拒 iff `llm ≠ accept`
- `llm_or_sim`(默认):拒 iff `(llm ≠ accept) AND (max_sim < sim_thr)`

### 3.6 ⚠️ 性能瓶颈(诚实)
se6 正确率仅 **15.1%**,系统远未可用。瓶颈经 T20 对抗审查 + 字段核查锁定(非单步消融因果):
1. **babble diar 误检**:babble ov0(本应单人)100% 检出 2 speaker。
2. **STNO target 帧清零**:`stno_target_ratio` babble 全档 = 0.00 → DiCoW 转写崩溃(英文幻觉/重复循环)。
3. **Whisper 带噪中文**:强制中文 token 起始压不住 decoder 后续漂移。
4. =0/=6 都救不了 babble diar 误检;se0 babble CER 较低仅因强降噪让幻觉更短。

## 4. 真实 A 集评测流程(A 到手后)

1. **准备 manifest**:把 A 集的 enrollment + recognition 配对写成 `--pairs` manifest(见使用说明 §3)。
2. **跑标准化推理**(在 L20 上):
   ```bash
   source code/setenv.sh && export HF_HUB_OFFLINE=1
   code/.venv/Scripts/python.exe code/submit_infer.py \
     --pairs <A集manifest.json> --out-dir A_out --device cuda:0
   # → A_out/result.json + A_out/timing.json
   ```
3. **算指标**(对照 A 集 ground truth):
   ```bash
   code/.venv/Scripts/python.exe code/eval_metrics.py \
     --result A_out/result.json --manifest <A集gt.json> --out A_eval.json
   ```
4. **报效率分**(L20 峰值显存 + overall_rtf,直接读 `timing.json`)。
5. **写测试报告**(分档:SNR / overlap / noise_type),与仿真 §3 对照。

## 6. 局限声明(诚实)

1. **仿真噪声为程序生成**(white/pink/babble),真实环境噪声谱可能不同 → `noise_classify` 谱平坦度阈值需在真实 A 集重新校准。
2. **TTS 合成 ≠ 真实远场**:mimo-tts 合成音频缺远场混响,真实数据会更难;可加 RIR 卷积改善(未做)。
3. **100% 重叠单通道死区**:两人同帧说话、单麦克风无方向信息,diarization 退化为把主导声归到多 speaker → 依赖**多通道空间分离**(DSENet/VSAEC,待通道数确认)。
4. **target-absent 72 条全为单说话人苏打**:真实比赛"≤2 说话人都非 target"更难,本结果证明机制成立但需扩。
5. **CAM++ 证伪基于 450 仿真**:T18 per-speaker 公平对照 sim 0.191 < wespeaker 0.218,但仅 SNR−5 档 CAM++ 略优;真实数据上需复测。
6. **babble 根因为字段关联非因果**:T20 "diar 误检+STNO 崩"基于 `speakers=[0,1] + stno_target=0 + 英文幻觉` 同现,因果链待单步消融确证(如 T19 langfix bug 待对抗审查才定论)。
