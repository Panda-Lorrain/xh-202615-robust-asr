# XH-202615 技术设计方案

> 美的集团发榜《复杂交互场景的抗干扰语音指令识别技术》(XH-202615)
> 本文档描述**实际已实现的方案**(截至 T20 / 2026-06-30),非理想态规划。理想态组件(CAM++/Personal VAD)在文中明确标注为"未用/证伪"。

---

## 1. 问题定义与评分

- **任务**:给定目标说话人的**唤醒音频(enrollment)**,在带噪(SNR −5~5 dB)+ 多人重叠(≤2 人,重叠率 0–100%)的**识别音频(recognition)**中,只转写目标说话人的指令、拒识非目标干扰。
- **评分维度**:
  - 目标 CER(字符错误率) — 40%
  - 拒识率(目标缺席时真实拒识 / 目标在场时误拒) — 40%
  - 推理效率(RTF、峰值显存,官方在 **L20 GPU** 上评测) — 20%
- **运行约束**:推理脚本需在 L20 上吃 enrollment + recognition → 输出标准化 `result.json` + `timing.json`。

---

## 2. 总体架构(实际实现)

**数据流**(已实现于 `code/submit_infer.py`,4 阶段 subprocess 编排):

```
recognition.wav
   │
   ├─[阶段0] noise_classify (谱平坦度)  ─→ 噪声类型 {white / pink / babble}
   │
   ├─[阶段1] SE 条件化降噪 (DeepFilterNet3)
   │         babble/white → atten=0 全力   pink → atten=6 温和(防过消除)
   │         ↓ denoised.wav
   │
   ├─[阶段2] enroll_infer
   │         ├─ DiariZen(wavlm-large) diarization → 各 speaker 时间段
   │         ├─ wespeaker 声纹(enrollment + 各 speaker,256d)→ 余弦匹配选 target_idx
   │         ├─ STNO mask(sil/target/nontarget/overlap 4 类,FDDT 仿射变换注入 encoder 每层)
   │         └─ DiCoW(Whisper-large-v3-turbo + FDDT)只转 target → transcript
   │
   ├─[阶段3] llm_reject (Qwen2.5-3B-Instruct)
   │         对 transcript 判 accept / reject(零样本,13 类拒识 schema + 自适应 CoT)
   │
   └─[阶段4] 融合 (llm_or_sim)
             rejected = (llm ≠ accept) AND (max_sim < sim_thr)   # 默认 sim_thr=0.2
             ↓
result.json  +  timing.json
```

**核心组件**(全部已开源、已跑通):

| 组件 | 实际用 | 备注 |
|---|---|---|
| 目标锁定 | **wespeaker-voceleleb-resnet34-LM**(256d,复用 `diar._embedding`) | 非 CAM++(T18 per-speaker 公平对照证伪,sim 0.191 < wespeaker 0.218) |
| 说话人分离 | **DiariZen**(diarizen-wavlm-large-s80-md) | 非理想态的 Personal VAD(未用) |
| TS-ASR 转写 | **DiCoW**(BUT-FIT/DiCoW_v3_2,Whisper-large-v3-turbo + FDDT) | 已修 `generation.py` language 强制死代码 bug(`apply_dicow_langfix.py`) |
| 语音增强 | **DeepFilterNet3**(条件化,8.7MB,纯 CPU) | 谱平坦度分桶,可部署 |
| 语义拒识 | **Qwen2.5-3B-Instruct**(6.17GB) | 救回声纹误拒 |
| 融合决策 | **llm_or_sim** | 声纹 + LLM 双信号 |

## 3. 各模块技术细节

### 3.1 目标锁定(wespeaker 声纹)

- **enrollment 声纹 + 各 speaker 声纹**统一复用 DiariZen 内部 `_embedding`(wespeaker backbone,256d),零额外模型加载。
- 抽 speaker 声纹时**优先 non-overlap 独占帧**(避开重叠污染),不足 0.3s 时 fallback 全 timeline;不足 1s 时 tile 补齐。
- **余弦匹配选 target**:`target_idx = argmax_i cos(enroll_emb, spk_emb_i)`,`max_sim` 作为"目标是否在场"的置信度。
- 可选 **enrollment 加噪增强**:enrollment emb = 干净 + 3 档加噪(10/5/0 dB)emb 均值 → 与带噪识别音频同分布,均 sim 提升 +59%。
- <!-- TODO: max_sim 阈值工作点 0.2(T20 经验最优,见 §5);干净场景同人 0.816 vs 不同人 0.035,阈值 0.5 居中;但带噪全集 sim 普遍退化(均 0.218),0.2 在题目分布更可用 -->

### 3.2 STNO 条件化转写(DiCoW + FDDT)

- **FDDT 仿射变换**将 `[silence / target / non-target / overlap]` 4 类 mask 注入 Whisper encoder **每一层**,对角约束 + `fddt_init=disparagement`(抑制式初始化) → 非目标帧被压到零输出。
- **STNO 验证(中英文均成立,T16)**:target 帧→正常转写;silence 帧→跳过;**non-target 帧→直接产出 0 字(内建拒识,非后处理)**。
- 基座 **Whisper-large-v3-turbo**(0.89G params,32 encoder + 4 decoder 层)。
- **已修 DiCoW `generation.py` language 死代码 bug**(T19):原代码 `language="zh"` 被静默忽略 → 退化音频上 `detect_language` 误检英文 → 450 集 90% 输出英文;补丁后 english 90%→72%,中文 39→125(3 倍)。补丁位置 `code/apply_dicow_langfix.py`,**HF 缓存清了需重打**。
- **STNO target 帧占比**(`stno_target_ratio`)作三路融合的第三信号,但 T19 验证其单独是**坏拒识信号**(可能选错 speaker → 高 stno 却非 target);拒识锚信号仍是声纹 `max_sim`。

### 3.3 SE 条件化(DeepFilterNet3)

- **谱平坦度**(spectral flatness)估每条识别音频的噪声类型:white(平坦度≈0.39)/ pink(≈0.11)/ babble(≈0.03),三类在 450 仿真集上**无重叠完全分离**。
- **条件化策略**(可部署,运行时纯谱分析无需 manifest):
  - babble / white → `atten_lim_db=0`(全力降噪)
  - pink → `atten_lim_db=6`(温和,防稳态噪声过消除反伤语音)
- 仿真集 SE 条件化 CER **2.82(相对 baseline 4.27 改善 −34%)**,且 diar-fail 数 33→0(pipeline 稳定性强信号)。
- <!-- TODO ⚠️: 谱平坦度阈值在 450 仿真 in-sample 达 99.78% 准确,真实噪声分布不同,准确率必然下降,阈值需在真实 A 集上重新校准;99.78% 不可直接外推到比赛 -->
- <!-- TODO: post-fix(T20 langfix 后)最优策略变为新 conditional(white,pink→6, babble→0)=2.609 / 全 se6=2.504 / 精细二维 2.022(oracle);与 pre-fix 旧 conditional 规则不同,真实 A 集到手后需重选 -->

### 3.4 LLM 语义拒识(Qwen2.5-3B-Instruct)

- **零样本** few-shot prompt,按出题方 #1 论文(arXiv:2512.10257 Midea AI Research)13 类拒识 schema + 自适应 CoT 四步(实体识别 / 动作识别 / 参数合理性 / 结论)。
- **34 条零样本测试集**:F1=**0.878** / Precision=0.783 / **Recall=1.0**(fn=0,该拒全拒)/ Accuracy=0.853。vs 全 reject 基线 F1=0.69(+0.19)。
- 5 个误拒(fp)都是合法复杂指令(窗帘拉一半/净化器睡眠/热水器 45 度/灯暖色最低/十分钟后提醒关火),prompt 调优 + 声纹融合可纠正。
- 作用:**救回声纹误拒**(声纹在带噪下退化会误拒合法 target);T19 实测在 450 集 LLM 拒了 99.8%(因 DiCoW 转写垃圾被正确判非指令),故 LLM 在"转写良好"时才能发挥救回价值。

### 3.5 融合(llm_or_sim)

- 默认策略 `llm_or_sim`:`rejected = (llm_verdict ≠ "accept") AND (max_sim < sim_thr)`。
- 可选 `sim_only` / `llm_only`(见 `submit_infer.py --strategy`)。
- **设计意图**:LLM 偏严(语义不合理即拒),声纹偏宽;两者**双否决** = 都说不像 target 才拒 → LLM 救回声纹误拒、声纹救回 LLM 误拒(转写垃圾时 LLM 会误拒)。
- T19 target 缺席集(72 条)验证:`sim_only(t=0.2)` 真实拒识率 **100%**,三路融合亦 100%(max_sim 均 0.026,全部正确拒)。

## 5. 当前性能(诚实,仿真 450 条)

> ⚠️ **以下全部为仿真数据(mimo-tts 合成 + 程序噪声),真实 A 集待评测。** 详见 [`测试验证方案.md`](测试验证方案.md)。

### 干净场景(无加噪)
| 重叠率 | target CER | 说明 |
|---|---|---|
| 0%(顺序) | **0.00** | 完美分离 |
| 50%(部分重叠) | **0.13** | "请把"误为"清白",基本可用 |
| 100%(完全重叠) | **1.00** | ❌ 单通道 diarization 死区,target 丢失 |

### 带噪全集(450 条,post-langfix)
| 配置 | overall CER | 正确率(CER<0.5) | 中文占比 | 说明 |
|---|---|---|---|---|
| se6 全集(=6 温和) | 2.504 | **15.1%**(68/450) | 58% | 最简,稳健 |
| 新 conditional(white,pink→6,babble→0) | 2.609 | — | — | 稳健推荐 |
| 精细二维(babble ov≤0.25→0,其余→6) | **2.022** | — | — | oracle 最优,过拟合风险 |
| se0 全集(=0 全力) | 3.542 | 7.8% | 30% | 最差 |

### 拒识侧
| 测试集 | 真实拒识率 | 说明 |
|---|---|---|
| target 缺席 72 条(sim_only t=0.2) | **100%** | max_sim 均 0.026,强阳 |
| LLM 零样本 34 条 | F1=0.878 / Recall=1.0 | 该拒全拒 |

### ⚠️ 性能瓶颈(诚实)
se6 正确率仅 **15.1%**,系统远未可用。根因(经 T20 对抗审查 + 字段核查锁定,非单步消融因果):
1. **babble diar 误检**:babble ov0(本应单人)100% 检出 2 speaker(把人声噪声当第 2 speaker)。
2. **STNO target 帧清零**:`stno_target_ratio` 在 babble 全档 = 0.00 → DiCoW 转写崩溃(英文幻觉/重复循环 596–714 字)。
3. **Whisper 带噪中文**:即使强制中文 token 起始,decoder 在 babble 退化信号上漂英文。
4. **=0/=6 都救不了 babble diar 误检**(se0 babble 也 speakers={2:30}+stno≈0.01);se0 babble CER 较低仅因强降噪让 Whisper 幻觉更短,非 diar 层改善。

## 6. 局限与改进路线

| 问题 | 改进方向 | 状态 |
|---|---|---|
| babble diar 误检 + STNO 崩 | babble 专用源分离 / STNO 构造绕过 diar / SE-DiCoW(enrollment 条件化) | 待 |
| 100% 重叠死区 | 多通道空间分离(DSENet/VSAEC,DOA 引导) | 待**通道数确认** |
| Whisper 带噪中文 | 中文家居微调(让 Whisper 在 babble+中文上鲁棒) | 待,重投入 |
| 真实环境噪声 | ESC-50/MUSAN/DEMAND 真实环境音替代程序噪声 | 下载失败,待 |
| TTS 分布 ≠ 真实远场 | RIR 远场卷积 + AISHELL 真实语料 | 待 |
| 边缘部署终态 | 量化/蒸馏/ONNX/流式 | 待确认目标硬件 |

### 待确认事项(影响路线)
- **测试集通道数**(单/多通道分水岭):决定 DSENet/KWS 空间路线能否用 → 向主办方确认或看 A 集数据。
- **目标硬件**(家电 MCU / 边缘网关 / 本地服务器):算力差几个数量级,决定轻量化力度。
