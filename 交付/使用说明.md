# 使用说明

> 标准化推理脚本 `code/submit_infer.py`:吃 enrollment + recognition → 输出 `result.json` + `timing.json`。
> 纯 stdlib subprocess 顶层编排,零侵入复用 `noise_classify` / `se_denoise` / `enroll_infer` / `llm_reject`。

---

## 1. 环境要求

### 硬件
- **GPU**:NVIDIA 8GB+(本机 RTX 4060 Laptop 8GB 已跑通;**官方评测在 L20 48GB**)
- 磁盘:权重 + 缓存约 15GB(全落 E 盘,禁 C 盘)

### 软件栈(3 个独立 venv,依赖冲突**不可合并**)

| venv | 路径 | 用途 | 关键依赖 |
|---|---|---|---|
| `.venv`(主) | `code/.venv/` | enroll_infer / noise_classify / DiariZen | torch 2.5.1+cu124 / transformers 4.42.4 / DiariZen |
| `.venv_se` | `code/.venv_se/` | se_denoise / DeepFilterNet3 | df-simple / DeepFilterNet3 |
| `.venv_llm` | `.venv_llm/`(项目根) | llm_reject / Qwen2.5-3B | transformers / vllm(可选) |

- **Python**:3.12(由 uv 管理,`uv run` 自动选版)
- **OS**:Windows 11 + Git Bash

### 环境配置(每次开新终端先 source)
```bash
source code/setenv.sh          # HF_HOME/UV_CACHE_DIR/代理 全重定向 E 盘
export HF_HUB_OFFLINE=1        # 离线模式,防 HF 直连卡死
```

## 3. 运行推理(`submit_infer.py`)

### 完整参数(读自 `argparse`)
| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `--pairs` | path | — | **二选一输入**:`[{enrollment, recognition}, ...]` manifest(一对一) |
| `--enrollment` | path | — | **二选一输入**:目标说话人参考 wav(需配 `--recognition-folder`) |
| `--recognition-folder` | path | — | 识别 wav 目录(配 `--enrollment`,一对多) |
| `--no-se` | flag | off | 跳过 SE 条件化降噪 |
| `--no-llm` | flag | off | 跳过 LLM 拒识 |
| `--sim-thr` | float | **0.2** | 声纹拒识阈值(T20 经验最优) |
| `--strategy` | enum | `llm_or_sim` | 融合策略:`llm_or_sim` / `sim_only` / `llm_only` |
| `--device` | str | `cuda:0` | 推理设备 |
| `--out-dir` | path | `code/submit_out` | 输出目录 |
| `--work-dir` | path | `<out-dir>/_work` | 中间产物 |
| `--limit` | int | 0(全部) | 只处理前 N 条(烟雾测试用) |

`--pairs` 与 `--enrollment` **互斥必填其一**;`--enrollment` 需配 `--recognition-folder`。

### 标准用法

**A. 一对多(同一 enrollment,一批识别音频)**:
```bash
source code/setenv.sh && export HF_HUB_OFFLINE=1
code/.venv/Scripts/python.exe code/submit_infer.py \
  --enrollment <enr.wav> \
  --recognition-folder <rec_dir> \
  --out-dir <out>
```

**B. 一对一(enrollment 与 recognition 配对,manifest)**:
```bash
code/.venv/Scripts/python.exe code/submit_infer.py \
  --pairs <manifest.json> \
  --out-dir <out>
```
`manifest.json` 格式:
```json
[{"enrollment": "path/to/enr.wav", "recognition": "path/to/rec.wav"}, ...]
```

### 降级 / 快测
```bash
# 跳 SE(纯 enroll+LLM):
... --no-se
# 跳 LLM(纯声纹拒识,显存不足时兜底):
... --no-llm
# 烟雾测试前 3 条:
... --limit 3
# 纯声纹策略:
... --no-llm --strategy sim_only
```

### 输出
- `<out>/result.json` — 每条结果(见 §4)
- `<out>/timing.json` — 端到端耗时(overall_rtf + phases + per_utt)
- `<out>/_work/` — 中间产物(noise_est / se_out / enroll_g* / llm_out,可删)

控制台末尾打印汇总:`N 条 (M 拒识) / overall_rtf=X (audio=Ys wall=Zs) / phases={...}`。

## 4. 输入输出格式

### `result.json`
```jsonc
{
  "task_id": "XH-202615",
  "generated_at": "2026-07-01T...",
  "config": {"se": true, "llm": true, "strategy": "llm_or_sim", "sim_thr": 0.2, "device": "cuda:0"},
  "n_utt": 3,
  "results": [
    {
      "utt_id": "rec_001",
      "enrollment": "path/enr.wav",
      "recognition": "path/rec_001.wav",
      "text": "请把客厅的空调温度调到二十六度",   // rejected=true 时为 ""
      "rejected": false,
      "score": 0.4401,                          // = max_sim
      "max_sim": 0.4401,
      "llm_verdict": "accept",                  // --no-llm 时为 null
      "noise_type": "white",                    // 谱平坦度估计
      "atten_lim_db": 0,                        // SE 降噪强度
      "diar_fail": false
    }
  ]
}
```

**字段语义**:
- `rejected=true` 时 `text=""`(拒识不输出转写);`score=max_sim`。
- `llm_verdict`:`accept` / `reject`(`--no-llm` 时为 `null`,融合退化为 `sim_only`)。
- `noise_type` / `atten_lim_db`:由阶段0 `noise_classify` 谱估计产生。**`--no-se` 时为 `null`**(SE 跳过 → 未估噪声,无 `atten_lim_db` 可填);启用 SE 时为 `"white"`/`"pink"`/`"babble"` 与 `0`/`6`。
- `diar_fail=true`:恶劣音频触发 pyannote reconstruct 边界 crash,enroll_infer 已 try/except 跳过(该条 `rejected=true`、`text=""`)。

### `timing.json`
```jsonc
{
  "device": "cuda:0",
  "n_utt": 3,
  "total_audio_sec": 8.2,
  "total_wall_sec": 35.7,
  "overall_rtf": 4.3537,        // = wall / audio
  "phases": {
    "noise_classify": {"wall_sec": 0.5},
    "se": {"wall_sec": 12.3, "n": 3},
    "enroll_diar_dicow": {"wall_sec": 20.1, "mean_rtf": 0.06},
    "llm": {"wall_sec": 2.8}
  },
  "per_utt": [{"utt_id": "rec_001", "audio_sec": 2.7, "wall_sec": null, "rtf": 0.058}, ...]
}
```

## 5. 常见报错与兜底

| 报错 | 原因 | 解决 |
|---|---|---|
| `diar crash: negative dimensions are not allowed`(pyannote reconstruct 边界) | 极端短音频/恶劣噪声 | `enroll_infer` 已 try/except 跳过该条(`diar_fail=true`),pipeline 不中断 |
| DiCoW 输出英文 / 中文转写垃圾 | HF 缓存被清,language 死代码 bug 复现 | 重打补丁:`code/.venv/Scripts/python.exe code/apply_dicow_langfix.py` |
| LLM venv 显存不足(OOM) | Qwen2.5-3B + DiCoW 同卡 8GB 紧张 | 用 `--no-llm` 兜底(退化为纯声纹 `sim_only` 拒识),或 `--device cuda:1` 分卡 |
| HF 直连卡死 | 网络问题 | `source code/setenv.sh` 设镜像 + 代理;权重已在 `E:/hf_cache`,设 `HF_HUB_OFFLINE=1` |
| `modelscope import` 挂起 | modelscope 1.37 + Win + torch2.5 兼容问题 | 不影响主线(主线用 wespeaker 非 CAM++);CAM++ 仅边缘部署备用 |
