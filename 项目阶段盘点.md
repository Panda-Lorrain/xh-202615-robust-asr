## 2. 每阶段详述

### W1 — 跑通 SE-DiCoW/DiCoW 开源 baseline（✅ 完成）

**做了什么**：从训练仓库误用纠正→锁定 `BUTSpeechFIT/DiCoW` 推理仓库→装 torch 2.5.1+cu124（避开清华源默认 CPU 版坑）→下 DiCoW_v3_2 权重（3.6G safetensors）→最小推理（手构造全-target STNO）测 RTF/显存→补 wespeaker + from_pretrained patch→**完整端到端 DiariZen diarization → STNO → DiCoW 真 target-speaker 转写**。

**产出物**：
- `code/minimal_infer.py` — 最小推理脚本（模型加载 + forward + generate + RTF）
- `code/setenv.sh` — C 盘重定向（HF_HOME/UV_CACHE_DIR/TORCH_HOME → E 盘，禁 C 盘）
- `code/DiCoW-inference/` — 推理仓库 + DiariZen submodule + pyannote-audio fork
- **5 个 fork patch**（见 PROGRESS.md「完整 pipeline 已跑通」）：① pyannote `__init__.py` namespace patch ② `__version__` 硬编码 ③ DiariZen `inference.py` from_pretrained 本地路径分支 + wespeaker local_files_only ④ 依赖声明 ⑤ wespeaker 权重（`pyannote/wespeaker-voceleb-resnet34-LM`，⚠️ VoxCeleb 带 x）
- **核心数字（实测，RESULTS.md）**：DiCoW_v3_2（Whisper-large-v3-turbo + FDDT），**params 0.89G**，30s 音频推理 1.73s，**RTF=0.058**，**峰值显存 2.13GB**（RTX 4060 8GB 够）
- **FDDT config 印证**：`use_fddt:true` / `fddt_init:"disparagement"`（抑制式初始化）/ `fddt_is_diagonal:true` / 四类 STNO 全开 / `encoder_layers:32 + decoder_layers:4`（turbo 配置）
- **完整 pipeline 实测（T14）**：EN2002a_30s 正确分离 **4 说话人并各自转写**（Speaker 3 = 主说话人，含 "yeah yeah but i do not know about you..." 全段），`PIPELINE_EXIT=0`

**遗留问题**：
- minimal 推理用手构造全-target STNO，已升级为完整 diarization（不构成遗留，已解决）
- 8GB 显存偏紧，长音频需分块/限 batch（监控显存）

**下一步**：把 fork patch 固化为 `code/DiCoW-inference/repro/`（apply_patches.sh + REPRODUCE.md，见设计文档 Part 2，幂等可重跑）

## 4. 当前能力边界

### ✅ 能做什么（已跑通）

- **TS-ASR 完整端到端 pipeline**：DiariZen diarization（wavlm-large）→ STNO mask → DiCoW_v3_2（Whisper-large-v3-turbo + FDDT）真 target-speaker 转写，PIPELINE_EXIT=0
- **英文多人分离转写**：EN2002a 4 说话人正确分离各自转写
- **中文 TS-ASR**：家居指令"请把客厅的空调温度调到二十六度"一字不差（CER=0）
- **重叠场景可控转写**：0% 重叠 target CER=0（完美）、50% CER=0.133（可用，"请把"→"清白"）
- **STNO 内建拒识**（中英文双重证实）：target 类→转、silence 类→跳、**non-target 类→0字拒识**（FDDT `fddt_init=disparagement` 抑制式初始化，非后处理）
- **效率基线**：RTF=0.058（远快于实时，L20 会更优），params 0.89G，GPU 峰值 2.13GB
- **评测/仿真/合成工具链**：eval_metrics.py（CER/RTF/拒识）、simulate_pipeline.py（SNR/重叠）、mimo_tts_zh.py（中文合成）
- **可复现**：5 个 fork patch 已核实真实存在，可固化（设计文档 Part 2）

### ⛔ 不能做什么（限制）

- **100% 完全重叠死区**：两人同帧说话、单麦克风无方向信息，diarization 退化为把主导声归到多个 speaker，target 指令完全丢失（CER=1.000）
- **极端场景 robustness**：完全重叠 + 短音频(2.7s) 触发 Whisper 幻觉（转出无关英文）；加噪 −5dB 时 diarization 维度 crash（`negative dimensions are not allowed`）
- **enrollment 自动锁定 target**：当前 pipeline 是"N 个 speaker 轮流各转"，target 是事后人工挑，**非模型按 enrollment 选定**（W3 待落地）
- **空间路线**：DSENet/KWS 空间/VSAEC 不可用——待通道数确认（且它们无预训练权重/未测 −5dB/DOA 在完全重叠同样歧义，不是银弹）
- **LLM 语义拒识**：W5 完全未启动，拒识侧仅有 STNO 机制层这一路

### ⚠️ 待确认项

1. **测试集单/多通道**（最关键，决定空间路线 DSENet/KWS 能否用，当务之急）—— 向主办方确认或看 A 集数据
2. **enrollment 格式**：整句唤醒词 / 任意片段？长度范围？（影响声纹策略 + SE-DiCoW 接入）
3. **标注粒度**：字级 / 句级？有无重叠区说话人标注？
4. **远场/近场**：识别音频是否含回声、混响？（决定是否需要 AEC，对应 VSAEC #7）
5. **指令集范围**：智能家居指令领域边界？（影响 LLM 拒识合理性判别标准）
