## M1. 声纹提取（enrollment → e_target）

### ① 原理
从目标说话人的唤醒音频中提取一个**与文本、时长无关的固定维向量**（speaker embedding / d-vector），作为后续所有模块"认人"的锚点。

### ② 关键数学
- **CAM++**（Context-Aware Masking）：骨干网络（ResNet/Conformer 变体）+ 上下文感知掩码 + 多层聚合，输出 **192 维** speaker embedding $e \in \mathbb{R}^{192}$。
- 声纹匹配用**余弦相似度**：$\text{sim}(e_a, e_b) = \frac{e_a \cdot e_b}{\|e_a\|\|e_b\|}$，或 PLDA 打分。
- 预处理：CMVN（倒谱均值方差归一化），分帧提 Mel/FBank 特征。

### ③ 设计选择（超短 enrollment 怎么办）
唤醒音频可能极短（<0.5s），CAM++ 直接提声纹会不稳。两条美的路线解决：
- **US-PVAD（#3）路线**：**不依赖外部 SV 模型**，用 DPRNN 的 RNN states 内生累积声纹（见 M2），省掉 15.5M 参数的 SV 模型。0.2s 参考即可。
- **ASE-PVAD（#2）路线**：仍用 CAM++ 192d，但从**混合识别音频里自动选目标说话人最活跃的关键帧**反哺 enrollment，把短注册的声纹"补强"。

### ④ 开源资产
- **CAM++**：ModelScope（`speech_campplus_sv_zh-cn_16k-common`）/ 3D-Speaker（`alibaba-damo-academy/3D-Speaker`），中文预训练权重，开箱即用。
- ECAPA-TDNN（SpeechBrain）：备选 SV 模型。

### ⑤ 工程注意
- 采样率对齐（CAM++ 多为 16k）；唤醒音频若为 8k 需重采样。
- 超短参考（<0.5s）建议走 US-PVAD 内生声纹路线，而非强行 CAM++。

### ⑥ 答辩问答
- **Q：为什么用 CAM++ 不用 ECAPA-TDNN？**
  A：CAM++ 是 ModelScope 中文声纹主力，中文家居场景预训练充分；且美的 ASE-PVAD（#2）直接基于 CAM++，路线一致、可复现其结果。ECAPA 作备选。
- **Q：唤醒音频只有 0.2s 怎么办？**
  A：走 US-PVAD（#3）的 DPRNN 内生声纹路线，0.2s 参考 REC 反超 baseline 2s 参考（62.74% vs 52.60%），加噪后优势更大。
- **Q：声纹会被噪声污染吗？**
  A：会。这正是 ASE-PVAD"从混合音频反哺"的价值——用更长时识别音频里的目标帧补强短注册声纹。

## M2. Personal VAD（识别音频 → 帧级 TSS/NTSS）

### ① 原理
普通 VAD 只区分"有声/无声"。**Personal VAD (PVAD)** 进一步区分"是谁的有声"——**三分类**：
- **NS**（Non-Speech）：静音/纯噪声
- **NTSS**（Non-Target Speaker Speech）：非目标说话人语音
- **TSS**（Target Speaker Speech）：目标说话人语音

PVAD 用目标说话人声纹 $e_{target}$ 作条件，逐帧判别这三类。

### ② 关键数学（US-PVAD，#3）
**声纹提取器（Ultra-Short Speech Extractor，基于 DPRNN）**：
- DPRNN = local BiGRU（intra-chunk）+ global BiGRU（inter-chunk），chunk=250，hop=125。
- **关键**：RNN states 记忆历史并持续更新 → 直接当说话人特征。
- **迭代 6 次**：每轮把参考语音再过一遍 DPRNN，取含 RNN states 的 slice，6 个 slice 拼成最终声纹 $E_{us} \in \mathbb{R}^{K \times 6R}$。

**PVAD Backbone**（4 层 Conformer）：
- 混合特征 → Conformer → $E_{concat}$
- $E_{us}$ 作 key/value、$E_{concat}$ 作 query → **Cross-Attention**
- CA 输出经 **FiLM** 调制：$\gamma \odot E_{concat} + \beta$
- 决策：$p = \text{FC}(\text{FiLM}(\text{Con}(F_{concat}),\ \text{CA}(\text{Con}(F_{concat}), E_{us})))$

**损失（Weighted Pairwise Loss）**：$w_{\langle tss,ns\rangle}=w_{\langle tss,ntss\rangle}=1,\ w_{\langle ns,ntss\rangle}=0.5$（目标类的判错惩罚最重）。

### ③ 设计选择（两条路线对比）
| | **US-PVAD（#3）** | **ASE-PVAD（#2）** |
|---|---|---|
| 声纹来源 | DPRNN RNN states（内生） | CAM++ 192d（外生） |
| 超短参考 | 6 次迭代累积 | 从混合音频选关键帧反哺 |
| 依赖 SV 模型 | ❌ 省 15.5M | ✅ CAM++ |
| 融合 | Cross-Attention + FiLM | FiLM |
| 参数量 | 1.93M（无 SV） | 1.1M PVAD + 15.5M SV |

> **本质**：#3 是"内生声纹"（网络自己记），#2 是"外生 + 自举"（外部声纹 + 混合音频补强）。两者可互补：超短参考走 #3，有足够参考走 #2。

### ④ 开源资产
- DPRNN（语音分离经典，开源）；Conformer；SpeechBrain ResNet34 SV（开源 baseline）。
- US-PVAD / ASE-PVAD 本体：论文未给官方代码（需自实现，但架构清晰）。

### ⑤ 工程注意
- PVAD 是**数据敏感**的——LibriSpeech（英文）仿真训练的模型，在真实中文家居场景有分布偏移，真实数据来了要重训。
- 三分类标注可由仿真拼接 + 强制对齐（MFA）自动生成。

### ⑥ 答辩问答
- **Q：PVAD 和普通 VAD 的区别？**
  A：普通 VAD 二分类（有声/无声），PVAD 三分类（NS/NTSS/TSS），多了一维"是不是目标说话人"。这一维直接服务"拒识非目标"。
- **Q：PVAD 输出怎么用？**
  A：两路：① 喂给 M3 转 STNO mask → 条件化 DiCoW（转写）；② 喂给 M6 作拒识的"声纹活动"证据（拒识）。
- **Q：为什么 PVAD 比 TSE（目标说话人提取）前端更优？**
  A：PVAD 是帧级轻量判别（1.93M），不显式分离音频，延迟和算力远低于 TSE 分离网络，利好 20% 效率分。这是出题方核心偏好。

## M3. STNO mask 与 FDDT（TSS/NTSS → Whisper 条件信号）

### ① 原理
DiCoW 不直接过滤音频，而是把"谁在何时说话"的**帧级概率**（STNO 四分类）通过 **FDDT（Frame-level Diarization-Dependent Transformations）** 注入 Whisper encoder 每层，让模型"知道"哪些帧是目标、哪些是非目标，从而只转写目标。

### ② STNO 四分类定义（关键）
- **S**（Silence）：静音
- **T**（Target）：仅目标说话人活动
- **N**（Non-target）：仅非目标说话人活动
- **O**（Overlap）：目标 + 非目标重叠

> **PVAD 三分类 → STNO 四分类的转换**（组合主线的胶水）：
> - TSS 且无他人 → T
> - TSS 且与他人重叠 → O（需 NTSS/能量交叉判别）
> - NTSS 且无目标 → N
> - NS → S
> 实现上可由 PVAD（TSS/NTSS）+ 普通 VAD 联合推出，或训练一个小转换头。

### ③ FDDT 关键数学（已核实原文 FDDT 2409.09543）
设 Whisper 第 $l$ 层 Transformer 的帧级输入 $Z^l \in \mathbb{R}^{d_m \times T}$。为每个 STNO 类 $i \in \{S,T,N,O\}$ 在每层学一个**仿射变换** $(W_i^l, b_i^l)$：

$$\tilde{z}_t^l = \sum_{i \in \{S,T,N,O\}} p_t^i \cdot (W_i^l z_t^l + b_i^l)$$

其中 $p_t^i$ 是第 $t$ 帧、第 $i$ 类的 STNO 概率。即**每帧的变换是四类仿射的凸组合**，权重就是 STNO 概率。

- **插入位置**：encoder **每层之前**；默认 **diagonal constraint**（$W$ 只学对角线，参数量小）。
- **Query-Key Biasing (QKb)**：DiCoW 额外在 attention 的 query/key 上加 bias，进一步引导模型聚焦目标帧。

### ④ 设计选择：抑制式初始化（最关键的巧思）
随机初始化 FDDT 会**破坏 Whisper 预训练表征**。两种初始化：
- **Identity Init（Non-Disturbing）**：$b=0,\ W=I$ → 启动时不改变 Whisper 任何输出，安全微调。
- **Suppressive Init**（默认）：$W_S, W_N$（静音/非目标）初始化为**对角值 ≈ 0.1 的对角矩阵**，$W_T, W_O$（目标/重叠）用单位矩阵。
  → **一开始就把非目标/静音帧"压下去"，目标帧"放出来"**，等价于把"拒识非目标"先验**焊进初始化**。收敛快、效果好。

> **学习率**：Whisper 主干 $2 \times 10^{-6}$，FDDT 新参数 $2 \times 10^{-4}$（新参数学得快，主干微调得慢，保护预训练）。

### ⑤ 开源资产
- **FDDT / DiCoW / SE-DiCoW 同仓库**：`github.com/BUTSpeechFIT/TS-ASR-Whisper` + HF 权重 `BUT-FIT/SE_DiCoW`。

### ⑥ 工程注意
- STNO mask 的质量直接决定 TS-ASR 上限——mask 越准，CER 越低。这是 M2 PVAD 必须做好的根本原因。
- diagonal constraint 是效率与效果的权衡点，答辩可能问。

### ⑦ 答辩问答
- **Q：为什么不直接把非目标帧的能量置零（input masking）？**
  A：FDDT 论文证明 input masking 会留下硬边界 artifact、伤害 Whisper；soft 的 FDDT 凸组合 + 抑制式初始化能在不破坏预训练的前提下条件化，效果显著优于硬掩蔽。
- **Q：抑制式初始化为什么有效？**
  A：它把"压低非目标"的先验编码进初始权重，模型从第一轮就在"拒识非目标"，而不是从随机扰动中学起——既保护 Whisper 预训练表征，又加速收敛。
- **Q：PVAD 三分类怎么变成 STNO 四分类？**
  A：PVAD 给 TSS/NTSS，加一个"重叠"判别（两路同时活动）即可推出 O 类。这是一个轻量逻辑/小转换头，不增加重计算。

## M4. DiCoW / SE-DiCoW（条件化 Whisper → 目标 transcript）⭐ CER 主力

### ① 原理（DiCoW 的核心创新）
**抛弃 speaker embedding，改用 diarization 输出（who-spoke-when）条件化 Whisper**。

### ② 为什么抛弃 speaker embedding（答辩必问，已核实原文）
传统 TS-ASR 用预训练 SV 提 speaker embedding 喂给 ASR，问题是：
- ASR 必须学一个"**speaker embedding → ASR 内部语音表征**"的跨空间映射。
- 这个映射要泛化到**未见过的说话人**，需要训练集含**大量不同说话人**的多人数据。
- 真实多人对话数据**稀缺**（几十小时级），映射学不好 → 泛化差。

DiCoW 的解法：直接用 **diarization 的时间-说话人活动概率**（STNO）条件化，**不需要学任何跨空间映射**——因为时间轴是共享的，"第 t 帧是目标"这个信息和 ASR 的帧表征天然对齐。→ **少数据即可微调，对未见说话人泛化好**。

### ③ SE-DiCoW 的增量（解决 DiCoW 的致命缺陷）
**DiCoW 的局限**：完全重叠区（两个说话人 100% 同说）的 STNO mask 对不同说话人**几乎相同**，但 transcript 应该不同 → 条件化歧义。

**SE-DiCoW 的解法（self-enrollment + cross-attention）**：
- 用 diarization 输出**自动定位对话中"目标说话人最活跃的片段"**作 enrollment segment。
- 该 segment 经 **cross-attention 注入 Whisper encoder 每层**，作固定条件（fixed conditioning）。
- 即在 STNO 之外，**额外加一路"目标说话人长时声纹"条件**，消解重叠歧义。
- 另改进：model initialization、training data segmentation、data augmentation。
- **结果**：EMMA MT-ASR benchmark，macro-averaged tcpWER **相对原 DiCoW 降 52.4%**。

### ④ 本题接入（SE-DiCoW 当 baseline 的关键改造）
SE-DiCoW 原本是 **self-enrollment**（从混合音频自动选片段）。本题**已经给定唤醒音频**，所以改造：
- 把 self-enrollment 的"自动选片段"替换为"**接受题目给定的唤醒音频**"作 enrollment segment。
- 架构完全兼容（cross-attention 接口不变），只改数据加载/推理逻辑。工作量 1–2 周。

### ⑤ 开源资产（最重要，有现成权重）
- 仓库：`github.com/BUTSpeechFIT/TS-ASR-Whisper`
- HF 权重：`BUT-FIT/SE_DiCoW`（基于 Whisper-large-v3-turbo）
- **中文适配缺口**：Whisper 支持中文，但 FDDT/cross-attention 未在中文多人数据训练 → 需少量中文多人数据微调（2–4 周）。

### ⑥ 工程注意
- 基座 Whisper-large-v3-turbo 较大，L20 推理 RTF 要测（关系 20% 效率分，见 M7）。
- STNO mask 是 DiCoW 的强依赖，mask 质量是 CER 上限。

### ⑦ 答辩问答
- **Q：为什么用 DiCoW 而不是传统 speaker-embedding TS-ASR？**
  A：见 ②——避免学习跨空间映射，少数据微调，对未见说话人泛化好。题目真实数据稀缺，这条优势直接命中。
- **Q：SE-DiCoW 解决了什么 DiCoW 解决不了的问题？**
  A：完全重叠区的条件化歧义——两个同说的人 STNO 几乎相同，但 transcript 不同。SE-DiCoW 加 self-enrollment cross-attention 提供长时声纹条件，tcpWER 降 52.4%。
- **Q：题目给定了唤醒音频，SE-DiCoW 的 self-enrollment 还有用吗？**
  A：self-enrollment 的"自动选片段"被题目给定的唤醒音频替代，但 cross-attention 机制保留——这是它的核心价值。我们改造的是输入来源，不是架构。

## M5. 后处理纠错（RASTAR 思路，可选，降 CER）

### ① 原理（#6 RASTAR，arxiv 2602.12287）
ASR 转录里的错误常集中在**命名实体 / 同音词**（如"美的"→"美迪"）。RASTAR 用 LLM 后处理纠错：
- **RLM**：非自回归整句改写做 NER（突破字符级对齐退化）
- **检索**：拼音编辑距离从实体库（16,168 实体）检索 top-3 候选
- **A-STAR**：自适应 CoT，按难度切 think/no-think；Formidable 档注入 hint + rejection sampling + DPO 自训练

### ② 效果
NE-CER 相对 DANCER 降 **17.96%**（AISHLL-1）/ **34.42%**（同音词集）；8B 模型 CoT token 减 21–30%（省 token = 省时间，利好效率）。

### ③ 本题用法
- 家居指令有大量专有实体（设备名、场景名），适合实体纠错。
- **可选模块**——先跑通 M1-M4 主线，CER 不达标再上 M5。

### ④ 答辩问答
- **Q：纠错会不会引入新错误？**
  A：会过纠。RASTAR 用检索候选约束 + DPO，降低过纠；本题应设置信度阈值，低置信不改。

## M6. LLM 拒识（三路融合）⭐ 拒识主力

### ① 原理（#1 Reject-or-Not，arxiv 2512.10257）
**拒识不是单纯"声纹不匹配就拒"**。美的 #1 指出：要判断"这条指令在智能家居场景**是否合理、是否针对当前设备**"——这是**语义层 + 声纹层**的联合判断。

### ② 三路融合拒识（本队设计）
| 路 | 信号 | 判据 |
|---|---|---|
| ① 声纹置信度 | 转写段声纹 vs $e_{target}$ 余弦相似度 | 低于阈值 → 拒（非目标说的） |
| ② LLM 语义合理性 | transcript 喂 Qwen-2.5-3B | "非合理家居指令 / 非针对本设备" → 拒 |
| ③ PVAD 检测 | 该段是否 TSS（目标活动） | 非 TSS → 拒 |

融合策略：加权投票 / 学习一个轻量融合头。三路互补——声纹挡"别人说的"，语义挡"不合理指令"，PVAD 挡"非目标活动段"。

### ③ 自适应 CoT（结合 #6）
拒识推理用自适应思维链：简单指令 no-think 直接判，复杂/边界指令 think 一下——省 token、利好效率。

### ④ 开源资产
- **Qwen-2.5-3B**（开源，美的 #1 同款基座）
- #1 拒识 benchmark + 方法（思路可迁移，需自建中文家居指令集）

### ⑤ 工程注意
- LLM 拒识是**独立模块**，与 CER/效率解耦，可独立优化迭代。
- "合理性"判别标准需从题目/数据归纳**指令集范围**（00 文档待确认第 5 项）。

### ⑥ 答辩问答
- **Q：拒识只靠声纹阈值不行吗？**
  A：不行。#1 论文证明：目标说话人也可能说"不合理指令"（如闲聊、误唤醒），干扰人也可能声音像。纯声纹阈值会有误杀和漏拒。必须加语义合理性判断。
- **Q：三路融合怎么定权重？**
  A：在验证集上学权重 / 融合头；或先用规则阈值快速 baseline，再优化。关键是三路信号互补，单路都有盲区。
- **Q：LLM 拒识会不会太慢（影响效率分）？**
  A：用 3B 小模型 + 自适应 CoT（简单指令不 think）；且拒识在转写之后，可与流式转写流水线重叠。效率可控。

## M7. 效率优化（TS-RNNT 形态 / 蒸馏 / 量化）⭐ 效率主力

### ① 目标
20% 效率分 = 在 L20 GPU 上 RTF（real-time factor）尽量小、延迟低。同分情况下效率决定排名。

### ② TS-RNNT 形态（已核实原文，arxiv 2209.04175，NTT/东京工大）
**把 TSE 功能吸收进 RNN-T，不单独做分离前端**：
- speaker encoder 从 enrollment 提 $h_{target}$（多层 NN + linear + time-average pooling）
- 在 RNNT encoder 第 $n$ 层用 **Hadamard 积**（逐元素乘）融合：$h_t^{(n)} \leftarrow h_t^{(n)} \odot h_{target}$
- **关键效率卖点**：$h_{target}$ 可**预计算（预注册）**，不进帧路径 → **计算复杂度与延迟 = vanilla RNNT**（RTF 0.40），相对级联 TSE+ASR **3 倍提速**；流式延迟 30–330ms。

> 这是效率维度最优的架构形态。代价：论文未开源权重，需 ESPnet 自实现或用 SpeechBrain recipe；且原验证在日文 CSJ，SNR 0–20dB（未覆盖 −5dB）。

### ③ Whisper 侧效率优化
- **基座选择**：Whisper-**large-v3-turbo**（已是蒸馏加速版，比 large-v3 快数倍）。
- **量化**：FP16 → INT8（bitsandbytes / GPTQ）/ INT4，显存与推理时间显著降。
- **蒸馏**：large-v3-turbo → 更小 student（可选，工程量大）。
- **批处理 + 流式 chunking**：长音频分块流式，降低首字延迟。

### ④ 效率 vs 精度权衡（答辩必问）
- DiCoW/SE-DiCoW 精度最高但模型大（不利效率）。
- TS-RNNT 效率最优但精度次之、未开源。
- **策略**：CER 主线用 SE-DiCoW（冲精度），效率冲刺时考虑 TS-RNNT 形态或 Whisper 量化。两套 baseline 并存，按评分加权择优或集成。

### ⑤ 答辩问答
- **Q：Whisper-large-v3-turbo 在 L20 上 RTF 多少？**
  A：需实测（W6）。turbo 已比 large-v3 快约 8 倍；FP16 下 L20 通常可近实时或更快，量化和批处理后可进一步降。（答辩给实测数）
- **Q：TS-RNNT 为什么能做到和 vanilla RNNT 一样快？**
  A：声纹 $h_{target}$ 预注册、预计算，只与 encoder 一层做一次 Hadamard 积（逐元素乘，几乎零开销），不进逐帧递归路径 → 复杂度与 vanilla RNNT 相同。
- **Q：效率和精度怎么权衡？**
  A：双 baseline 策略——SE-DiCoW 保精度上限，TS-RNNT/量化保效率下限，按 40/40/20 加权和实测择优。

## 真实数据迁移指南（数据下发后怎么做）

> 核心结论：**工程资产 100% 复用，只做"微调 + 校准 + 适配"三件事，不返工。**

| 资产类型 | 迁移方式 | 是否浪费 |
|---|---|---|
| W1 推理 pipeline / 环境 | 直接复用 | ✅ 不浪费 |
| W2 仿真造数**脚本** | 直接复用，参数据真实分布**校准** | ✅ 不浪费 |
| W2 仿真**数据** | 从"主力"降级为"增强补充"（补极端条件、防过拟合） | ✅ 长期有价值 |
| W3 PVAD 架构/训练代码 | 复用，模型**用真实数据重训** | ⚠️ 权重重训（预期内） |
| W4 声纹接口 | 直接复用（CAM++ 预训练通用） | ✅ 不浪费 |
| W5 LLM 拒识框架 | 直接复用，prompt 据真实指令**微调** | ✅ 不浪费 |
| W6 评测/效率工具 | 直接复用 | ✅ 不浪费 |
| W7 接入改造 | 直接复用（架构改造与数据无关） | ✅ 不浪费 |

**三件事**：① 各模块真实数据 fine-tune（有权重初始化，收敛快）；② 仿真参数据真实分布反推校准；③ 接口层适配（enrollment 格式 / 采样率 / 标注格式）。

**唯一风险**：仿真假设与真实数据**结构性不符**（enrollment 是整句还是片段、远场还是近场、字级还是句级标注）。缓解：按最宽泛假设设计接口，把不确定性吸收在接口层。→ 务必第一时间确认 00 文档第九节的 5 个待确认项。

## 答辩核心问答汇总（高频尖锐问题）

### 架构与选型类
1. **Q：为什么用组合主线而不是端到端单一模型？**
   A：组合主线每环节都有开源成熟资产 + 美的自研偏好，可独立优化、可解释、可降级；端到端是上限赌注（见 02 文档），风险高，作为探索分支并行。
2. **Q：和别的队用同样开源方案，你的优势？**
   A：数据增强深度（D1）+ 中文家居微调（D2）+ 三路融合拒识（D4）+ 效率优化（D5）+ 端到端联合训练（D3 上限）。见 00 文档差异化矩阵。
3. **Q：单通道 / 多通道分别怎么办？**
   A：组合主线通道无关，单通道即可跑通（预案 A）；多通道在前端叠加 DSENet/VSAEC 空间提取（预案 B）。见 00 文档第四节。

### 技术深度类
4. **Q：FDDT 的凸组合为什么优于硬掩蔽？** → M3 ⑦
5. **Q：DiCoW 为何抛 speaker embedding？** → M4 ②
6. **Q：抑制式初始化的原理？** → M3 ④
7. **Q：PVAD 三分类怎么转 STNO 四分类？** → M3 ②
8. **Q：TS-RNNT 为何 RTF 与 vanilla 相同？** → M7 ②
9. **Q：SE-DiCoW 的 self-enrollment 在本题如何改造？** → M4 ④

### 评分策略类
10. **Q：40% CER 怎么保证？** A：SE-DiCoW 冲精度 + 中文微调（D2）+ 数据增强（D1）+ 可选 RASTAR 纠错（M5）。
11. **Q：40% 拒识怎么保证？** A：三路融合（声纹+语义+PVAD，M6），独立模块可单独优化。
12. **Q：20% 效率怎么保证？** A：turbo 基座 + 量化 + TS-RNNT 形态（M7），双 baseline 权衡。

### 鲁棒性类
13. **Q：−5dB 极端噪声怎么办？** A：数据增强覆盖（D1）+ Whisper 鲁棒性 + 可选 VPIDM 扩散增强（权衡效率）。
14. **Q：超短 enrollment（0.2s）怎么办？** A：US-PVAD 内生声纹路线（M1③/M2③）。
15. **Q：100% 完全重叠怎么办？** A：SE-DiCoW 的 cross-attention 专门解决重叠歧义（M4③）。
